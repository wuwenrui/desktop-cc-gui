## 功能说明

本 Skill 用于处理**法律意见书/法律事务记录表**出具的法律事务工作流。以律师身份，为客户特定法律问题出具专业、客观、有依据的书面法律意见，为客户决策提供参考，同时做好律师自身风险防护。

---

## 一、核心工作流与逻辑

### Step 1：了解项目背景（阅读资料）

- **操作**：进入工作区目录，阅读全部相关文件（合同、函件、批复、照片等）
- **检查项**：
  - [ ] 合同文本（当事人、标的、价款、权利义务、违约责任、争议解决、解除条款）
  - [ ] 往来函件/内部审批件
  - [ ] 行政批复/许可文件
  - [ ] 客户提出的问题原文
- **产出**：对案件事实的完整梳理，包括各方主体、法律关系、已发生事实、待决策事项
- **要点**：弄清楚客户到底遇到了什么问题、想要什么结果、谁是相关方

### Step 2：整理客户需求与核心法律问题

- **操作**：基于阅读材料，提炼客户的核心诉求和需要解决的法律问题
- **检查项**：
  - [ ] 客户问的到底是什么（表面问题 vs 深层问题）
  - [ ] 哪些是法律问题、哪些是商业/管理问题
  - [ ] 每个法律问题对应的法律关系和法律主体
  - [ ] 需要回避的陷阱（如"客户要求违法操作"）
- **要点**：在客户可能"问错了问题"的情况下，需要帮客户重新定义问题

### Step 3：研究分析核心问题（内化分析 + 联网检索）

- **操作**：
  1. 先凭自身法律知识进行初步分析，形成初步判断
  2. **联网检索验证**：使用 WebSearch 检索相关法律法规、司法解释、部门规章、地方性规定、最新政策文件、典型案例
  3. 检索要点：
     - 核心法条原文及最新修订版本
     - 司法解释和裁判规则
     - 地方性规定（如涉及山西省的，优先检索山西省/太原市地方规定）
     - 最新政策动态（注意文件时效性）
     - 类似案例的裁判观点（最高院指导案例、典型案例）
     - 学术理论和实务观点
  4. 联网检索验证后，修正完善分析观点
- **检查项**：
  - [ ] 是否有直接适用的法条（上位法优先）
  - [ ] 是否存在法律冲突或空白
  - [ ] 是否有地方性特殊规定
  - [ ] 检索结果是否权威可靠
  - [ ] 观点是否有充分的法律依据支撑
- **要点**：观点必须来源于法律、政策或判例/理论，不能凭空杜撰

### Step 4：形成成熟观点

- **操作**：在充分研究基础上，形成清晰、有层次、有依据的法律分析意见
- **观点要求**：
  - 立场客观但有立场——为客户提供专业意见的同时守住法律底线
  - 每个观点必须有法律依据（法规名称+条款号+条文内容）
  - 对于法无明文规定或存在争议的问题，说明不同观点并给出倾向性意见
  - 预测可能的风险和后果
- **结构**：
  ```
  一、【核心问题1】
  （一）法律依据
  （二）法律分析
  （三）结论
  二、【核心问题2】
  ...
  ```

### Step 5：出具正式书面意见

- **操作**：按照客户要求的格式（通常为docx）出具正式法律意见书
- **常规结构**（法律事务记录表格式）：
  1. 律所名称 + 文件名称
  2. 客户信息表头（客户号、日期、承办律师、客户名称、联系人、业务类型）
  3. **咨询问题**（复述客户问题概述）
  4. **律师意见**（核心内容）
  5. **备注**（交付方式、限制性说明）
  6. **盖章**
- **律师意见书写要点**：
  - 标题分明：一、二、三、（一）（二）（三）
  - 法律分析在前，结论在后
  - 重要法条要引用原文
  - 建议要有可操作性
  - 风险提示要到位
  - 注意保护律师自己（不要承诺结果、要说明意见限制条件）

### Step 6：风险提示与自我保护

- **在每个意见书的末尾或适当位置进行风险提示**：
  - 说明意见基于客户提供的事实材料
  - 说明意见仅基于现行法律法规
  - 说明意见仅供客户内部决策参考
  - 对合同中未见的文件/未提供的信息进行保留
  - 对政策可能的变化进行提示

## 三、常见意见书类型及结构要点

### 3.1 法律咨询类（法律事务记录表）
适合：客户提出具体法律问题，律师书面解答
- 表格形式，简洁明了
- 重点在律师意见栏
- 一般不超3000字

### 3.2 专项法律意见书
适合：重大事项、专项问题
- 正式函件格式
- 含"出具依据与声明"部分
- 含事实概述、法律分析、结论与建议
- 落款律所盖章

### 3.3 风险提示函
适合：客户正在或拟进行某项业务，提示潜在法律风险
- 以风险为导向
- 重在风险说明和防范建议
- 语气建议性而非绝对

## 四、注意事项

1. **资料核实**：客户提供的材料可能存在不完整或不准确的情况，应予以说明
2. **立场问题**：既是律师又是顾问单位，要在客户利益和法律底线之间平衡。违法的事情不能帮
3. **语言风格**：专业、清晰、逻辑性强，但避免过于学术化，让客户能读懂
4. **记录保留**：保留工作底稿、检索记录、起草过程，备查
5. **时效性**：注意法规的时效性，引用最新版本
6. **联网优先**：涉及具体法规条款和政策时，必须联网检索确认，不能依赖训练数据
7. **交付确认**：意见书交付后，建议确认客户收到并理解主要内容
