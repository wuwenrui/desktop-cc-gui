## 功能说明

本 Skill 用于辅助撰写**不良资产包收购法律尽职调查报告**。

---

## 阶段零：开工前确认（必须全部通过后才能进入下一阶段）

收到出报告请求后，不要立即开工。依次完成以下三步确认，缺一不可：

### 确认一：项目文件夹路径

明确询问用户：**"请提供本次尽调项目的工作文件夹路径（绝对路径）。"**

等待用户回复到绝对路径后，再进入下一步。如果用户只给了项目名称或模糊位置，主动列出当前工作区下可能匹配的文件夹供用户选择。

### 确认二：模板确认

1. 先在项目文件夹及其上级目录中自动搜索格式模板文件（关键词：`格式模板`、`定稿`、`尽调`、`.docx`）
2. 如果只找到一个模板，告诉用户：**"本次将使用以下模板：【文件名】。是否继续使用该模板，还是需要更换？"**
3. 如果找到多个可能的模板，列出所有候选，请用户选定一个
4. 如果未找到模板，主动询问用户提供模板文件路径
5. 用户确认模板后，**立即阅读并学习模板**的权利义务声明、正文结构、字体字号、段落格式等。特别注意模板中的**表格数量、列数、表头**，后续生成时需保持完全一致
6. 学习完毕后告知用户：**"模板已学习完毕，将按此格式出具报告。"**

### 确认三：企业报告提醒

明确提醒用户：**"为提升报告工商信息、被执行信息等内容的完整度，请将各债务人和保证人企业的企查查/天眼查/启信宝专业报告（Word或PDF格式）放入项目文件夹下的子目录中（如 `企业报告/`）。如暂未准备好，可先跳过，但报告中相关部分将标注为'待拿到贷款档案后核实补充'。是否已准备好企业报告？"**

### 确认四：债权清单Sheet确认（重要！）

Excel债权清单可能存在多个Sheet（如汇总Sheet、分类Sheet等），**必须明确请用户确认使用哪个Sheet**。打印Sheet名称和各Sheet债务人数量，让用户选择。

> ⚠️ **教训**：初次处理时误将63户（全部Sheet）当作债务人，实际仅23户（指定Sheet）。每户的诉讼状态、保证人、抵质押物均从选定Sheet提取。

## 阶段一：资料梳理

### 第一步：识别债权清单

在项目文件夹中搜索 Excel 文件（`.xlsx`），自动识别债权清单：
- 读取所有 Sheet，逐个分析列名（关键词：债务人、本金、利息、保证人、抵押、诉讼、判决等）
- 存在多个 Sheet 时，结合 Sheet 名称和列结构判断哪个是本次转让清单
- **打印识别到的 Sheet 名称和债务人数量，请用户确认用哪个 Sheet**
- 提取每户债务人的：名称、债权本金、利息、费用、债权合计、保证人、抵质押物、诉讼状态

### 第二步：建立名称映射

企查查报告的文件名可能与债权清单名称不一致（如"{{公司}}"vs"{{公司}}"）。需建立映射表：

```python
NAME_MAP = {
    '报告文件名': '债权清单名称',
    '{{公司}}': '潞城宏远化工公司',  # 企业更名/别名
}
```

对于名称不一致且无法确认的，在报告中标注：
```
（注：企业报告名称为"XXX"，与债权清单记载不一致，待贷款档案核实）
```

## 阶段二：生成报告初稿

### 关键原则：**保留模板的表格式排版，绝不删除模板表格**

> ⚠️ **重要教训**：模板中的企业信息是表格式展现（docx Table），不是纯文本段落。生成报告时**必须使用表格**，不得删除模板表格后替换为纯文本段落。用户明确强调："尤其不能摈弃表格式表达"。

### 第三步：创建报告文件

1. 复制格式模板，按命名规则重命名：`【初稿】项目方名称-不良资产包收购法律尽调报告+YYYY.M.D.docx`
2. 存入项目文件夹

### 第四步：保留封面和声明

- 封面、标题、声明与陈述部分原封不动保留模板格式
- 仅修改项目名称和委托方名称（根据项目实际情况）
- **删除旧正文（第二部分）**，从空白开始添加新内容

### 第五步：逐户生成正文（表格式）

为每户债务人生成完整的六部分结构。各部分内的表格格式必须与模板一致。

**（一）基础债权**
- 引导句：`"根据【项目方】提供的资料显示：【项目方】向XX公司（以下简称"XX"）发放X笔借款，借款的具体情况如下："`

- 表头：仿宋12pt加粗，无彩色底纹
- 金额使用千分位逗号分隔格式

**（二）基础债权审查**
- 1.基础合同审查：填入尚欠金额，注明"以贷款档案资料及借款合同约定为准"
- 2.诉讼时效法律审查：根据诉讼状态判断
- 3.担保审查：区分保证担保和抵押担保，列明具体内容

**（三）主体资格审查**
- 1.借款人基本情况
  - **（1）工商登记情况**：3列Table Grid表格（字段名 | 企查查值 | 国家企业信用信息公示系统值），表头仿宋12pt不加粗，无底纹，边框thinThickSmallGap/sz=12。字段按模板顺序：借款人名称→曾用名→法定代表人→注册资本→实缴资本→状态→注册时间→营业期限→核准日期→统一信用代码→工商注册号→组织机构代码→纳税人资质→人员规模→参保人数→所属行业→曾用名→登记机关→经营范围→注册地址。
    - **⚠️ 股东信息和董监高嵌入在工商登记表内部**：模板的工商登记表是综合表格，末尾嵌入两个子节：
      - "董事、监事、高级管理人员信息"节：子表头"名称|名称|职务"，逐行填入从企查查 Table 4（主要人员）提取的数据
      - "股东信息"节：子表头"股东|股东|认缴出资（万元）"，逐行填入从企查查 Table 3（股东信息）提取的数据
  - **（2）对外投资情况**：网格型3/4样式（7列：序号|被投资企业名称|法定代表人|投资数额（万元）|持股比例（%）|注册时间|状态），表头ADB9CA+tint=66加粗。
    - **引导句（有数据）**：`（2）对外投资情况：企查查显示XX公司对外投资N家企业，具体情况如下：`
    - **引导句（无数据）**：`（2）对外投资情况：无。`
    - **⚠️ 仅读取当前对外投资**：企查查报告中 Table 5（被投资企业）是企业的当前对外投资。Table 12-14（担任法定代表人/对外投资持股/任职信息）是**法定代表人个人的**其他任职和投资，不属于企业对外投资，不应混入
    - **⚠️ 数据源列映射**：企查查对外投资表列：`序号|被投资企业名称|状态|持股比例|认缴出资额/持股数|认缴出资日期|所属省份|所属行业`。映射到模板列：被投资企业名称→被投资企业名称（直接）；持股比例→持股比例(%)(去百分号)；认缴出资额/持股数→投资数额(万元)(去"万元"后缀)；状态→状态（直接）；法定代表人→**待查**（企查查无此列）；注册时间→取"认缴出资日期"或**待查**
  - **（3）动产抵押情况**：5列 序号|登记编号|抵押权人|主债权数额（万元）|登记日期，表头ADB9CA+tint=66加粗。
    - **引导句（有数据）**：`（3）动产抵押情况：企查查显示XX公司N笔动产抵押，情况如下：`
    - **引导句（无数据）**：`（3）动产抵押情况：无。`
  - **（4）股权质押情况**：6列 序号|登记编号|出质人|质权人|出质股权数额（万元）|登记日期，表头ADB9CA+tint=66加粗。
    - **引导句（有数据）**：`（4）股权质押情况：企查查显示XX公司N笔股权出质，情况如下：`
    - **引导句（无数据）**：`（4）股权质押情况：无。`
  - **（5）司法拍卖情况**：4列 执行案号|标的物|执行法院|当前阶段，**无序号列**，表头D9E2F3加粗（非ADB9CA）。
    - **引导句（有数据）**：`（5）司法拍卖情况：通过企业信息查询工具-企查查，查询到XX公司司法拍卖信息如下：`
    - **引导句（无数据）**：`（5）司法拍卖情况：无。`
  - **（6）被执行案件情况**：总领段 `（6）被执行案件情况：`，其下四个子类各自独立成段，**不用"一、二、三、四"编号**：
    - **被执行人（无数据）**：`通过企业信息查询工具-企查查，未查询到XX公司作为被执行人的案件信息。` + 表格填 `1|无||||`
    - **被执行人（有数据）**：`通过企业信息查询工具-企查查，查询到XX公司作为被执行人的案件信息如下：` + 表格（5列：序号|案号|执行标的（元）|执行法院|立案日期），**有合计行**（执行标的额求和，前四空白写""）
    - **终本（无数据）**：`通过企业信息查询工具-企查查，未查询到XX公司终本执行案件信息。` + 表格填 `1|无||||||`
    - **终本（有数据）**：`通过企业信息查询工具-企查查，查询到XX公司终本执行案件信息如下：` + 表格（7列：序号|案号|执行法院|立案日期|终本日期|执行标的(元)|未履行金额(元)），**有合计行**（前四空白""，第5列"合计"，执行标的+未履行分别求和）

    - **失信（有数据）**：`通过企业信息查询工具-企查查，查询到XX公司被列入失信被执行人的信息如下：` + 表格（6列：序号|案号|执行法院|执行依据文号|履行情况|立案日期），**无合计行**
    - **限高（无数据）**：`通过企业信息查询工具-企查查，未查询到XX公司被采取限制高消费措施。` + 表格填 `1|无|||||||`
    - **限高（有数据）**：`通过企业信息查询工具-企查查，查询到XX公司被采取限制高消费措施的信息如下：` + 表格（8列：序号|案号|执行法院|限制法定代表人|申请人|涉案金额（元）|立案日期|发布日期），**无合计行**
    - **⚠️ 无被执行人的总段处理**：当四类均无数据时，总引导句统一写为：`通过企业信息查询工具-企查查，未查询到XX公司作为被执行人、终本执行、失信被执行人或被限制高消费的相关信息。`
    - **⚠️ 无被执行时的表述**：当企查查有查询结果（有表格），不能用"未查询到"——仅当企查查真的没有相应类别数据时，才用"未查询到"。有数据时用"查询到"+"信息如下"
  - 表头样式：全部表头使用 仿宋12pt加粗 + 垂直居中。底纹：执行案件/对外投资/股权质押/动产抵押表使用 ADB9CA+themeFill=text2+themeFillTint=66；司法拍卖表使用 D9E2F3（无themeFill/tint）。
  - 金额格式：千分位逗号分隔，两位小数（如 `21,689,603.00`），金额不带"元"或"万元"后缀（列标题已说明单位）

### 独立文档格式："对外投资+动产抵押+股权质押+司法拍卖+被执行案件信息"文稿

当需要为单个企业生成独立的（2）-（6）小节文稿时，按以下结构组织：

**段落顺序**（每个子节独立成段）：
1. `（2）对外投资情况：企查查显示XX公司对外投资N家企业，具体情况如下：`（或"无。"）
2. [对外投资表/无]
3. `（3）动产抵押情况：无。`（或"企查查显示XX公司N笔动产抵押，情况如下：" + 表）
4. `（4）股权出质情况：无。`（或"企查查显示XX公司N笔股权出质，情况如下：" + 表）
5. `（5）司法拍卖情况：无。`（或"通过企业信息查询工具-企查查，查询到XX公司司法拍卖信息如下：" + 表）
6. `（6）被执行案件情况：`
7. `通过企业信息查询工具-企查查，[未]查询到XX公司...` × 4类
8. [四类执行信息表]

**⚠️ （2）-（5）空数据格式铁律**：引导句和内容必须在**同一段落**，以冒号分隔。无数据时直接写 **"无。"**（引导句 + 冒号 + "无。" 拼接），不得用长句"通过企业信息查询工具-企查查，未查询到..."。仅（6）的四种子类无数据时保留完整句子格式。

### 企查查数据提取与列映射技术

**对外投资（T5）→ 模板7列**：
```
企查查列: 序号|被投资企业名称|状态|持股比例|认缴出资额/持股数|认缴出资日期|所属省份|所属行业

```

### 缺失信息用语规范

**"没有就是无"**：对于企查查等工具中确实不存在的信息（如无失信记录、无对外投资、无动产抵押等），直接写 **"无"**，不写"暂未查询到相关信息"。

对于确实需要从贷款档案中核实的信息，标注为 **"待查"**：
- 工商信息缺失且未查到企业报告 → "（1）工商登记情况：待查。"
- 保证人信息缺失 → "基本个人信息：XX。具体信息待补充。"
- 涉诉情况缺失 → "具体涉诉情况待补充贷款档案资料后调查。"

### 数据提取原则（重要！）

**"见企查查报告"是无效输出**：提取数据的目的是让报告自包含，不得出现"见企查查报告"这类需要用户去翻查其他文件的表述。

每条信息要么填入具体数据，要么根据实际情况写"无"或"待查"：
- 对外投资 → 从报告表格提取后填入投资列表，无则"无"
- 动产抵押/股权质押/司法拍卖 → 从报告段落提取数量，为0则"无"，有数据则展示
- **股东信息 → 从"股东信息"表格提取姓名和认缴出资额，无则"无"**（不得残留模板占位文字）
- **主要人员 → 从"主要人员"表格提取姓名和职务，无则"无"**（不得残留模板占位文字）

#### ⚠️ 股东信息表识别与提取

企查查报告中"股东信息"表格位于工商信息表之后，是一个独立表格，**必须识别并提取**，否则报告中股东信息会残留模板占位数据。

**表格识别**：企查查报告的"股东信息"表通常以段落标题"股东信息"开头，紧接着是一个标准表格。识别方法——遍历文档所有表格，定位前一个段落文本为"股东信息"的表格：

```python
def find_shareholder_table(doc):
    \"\"\"定位股东信息表。遍历文档元素，找到'股东信息'段落后的第一个表格。\"\"\"
    for i, elem in enumerate(doc.element.body):
        if elem.tag.endswith('}p'):
            # 检查段落文本是否为"股东信息"
            texts = [node.text for node in elem.iter() if node.text]
            para_text = ''.join(texts).strip()
            if para_text == '股东信息':
                # 向后查找紧随的第一个表格
                for j in range(i + 1, len(doc.element.body)):
                    next_elem = doc.element.body[j]
                    if next_elem.tag.endswith('}tbl'):
                        # 将 lxml 元素映射回 python-docx Table 对象
                        for table in doc.tables:
                            if table._tbl == next_elem:
                                return table
    return None
```

**列结构与提取**：股东信息表通常为多列（5-8列），列标题包含：
- `股东名称` / `发起人` / `投资人名称`
- `认缴出资额` / `认缴出资额（万元）` / `认缴出资额(万元)`
- `持股比例` / `出资比例`
- `认缴出资日期`

提取逻辑——从表格正文行（跳过表头）逐行读取，过滤空行：

```python
def extract_shareholders(doc):
    table = find_shareholder_table(doc)
    if not table:
        return []
    
    # 定位表头行，确定列索引
    header_row = None
    col_name = None
    col_capital = None
    for ri, row in enumerate(table.rows):
        cells_text = [c.text.strip() for c in row.cells]
        if '股东名称' in cells_text or '发起人' in cells_text:
            header_row = ri
            # 确定关键列索引
            for ci, ct in enumerate(cells_text):
                if ct in ('股东名称', '发起人', '投资人名称'):
                    col_name = ci
                if ct and ('认缴出资额' in ct or '认缴出资' in ct):
                    col_capital = ci
            break
    
    if col_name is None:
        return []
    
    shareholders = []
    for ri in range(header_row + 1, len(table.rows)):
        cells = table.rows[ri].cells
        name = cells[col_name].text.strip() if col_name < len(cells) else ''
        capital = cells[col_capital].text.strip() if col_capital is not None and col_capital < len(cells) else ''
        if name:
            shareholders.append({'name': name, 'capital': capital})
    
    return shareholders
```

**报告填充格式**：将提取结果拼接为报告文本：
```python
def format_shareholders(shareholders):
    if not shareholders:
        return '无'
    parts = []
    for s in shareholders:
        cap = f'认缴{s[\"capital\"]}万元' if s['capital'] else ''
        parts.append(f'{s[\"name\"]}（{cap}）' if cap else s['name'])
    return '，'.join(parts)
```

#### 提取注意事项

1. **表格不一定存在**：部分企查查报告可能缺少"股东信息"或"主要人员"表格（报告导出选项未勾选），此时对应字段写"待查"
2. **区分多个同名段落**：文档可能多次出现"企业名称"等文本，不要误匹配——必须精确匹配段落全文"股东信息"、"主要人员"
3. **表格定位不依赖顺序**：不同版本企查查报告各表格排列顺序可能不同，必须通过段落标题文本定位，不能依赖"第N个表格"这类硬编码
4. **表头行不固定**：不要假定表头在第1行——表头行可能因合并单元格在第2行，需遍历查找含关键列名的行
5. **空值处理**：股东名称存在但认缴出资金额为空（或为"-"）→仅写股东名称，不写"认缴万元"；职务为空→仅写姓名

- 统一社会信用代码、法定代表人、注册资本、成立日期
- 经营范围、登记状态（存续/注销/吊销）
- **股东信息**：从`extract_shareholders()`获取，按`format_shareholders()`格式化填充。替换模板中股东占位文字。如无报告或提取为空→写"无"
- **主要人员（董监高）**：从`extract_personnel()`获取，按`format_personnel()`格式化填充。替换模板中董监高占位文字。如无报告或提取为空→写"无"
- 对外投资/动产抵押/股权质押/司法拍卖 → 从报告段落提取数量，为0则"无"，有数据则展示
- 被执行案件情况：
  - 有失信数据 → 5列失信表（序号|案号|执行法院|涉案金额|履行情况）
  - 有终本/限高 → 文字说明条数
  - 无数据 → "暂未查询到XX失信被执行人记录"

> ⚠️ **模板克隆法下的股东/董监高陷阱**：使用`deepcopy`克隆模板债务人节时，模板中的股东姓名、认缴出资金额、董事/监事/高级管理人员姓名和职务等占位文字会被原样复制。必须在克隆后逐一替换为提取到的真实数据。这是最容易被遗漏的错误——生成报告后逐户抽查确认股东和主要人员信息不是模板占位文字。

### 名称映射数据合并

同一企业可能存在多笔债权（如山西德玺有建行/中行两笔），企查查数据只有一份，需建立合并匹配：
```python
def match_extracted(name, extracted):
```

- 文件格式：`.docx`
- 正文字体：仿宋 12磅（四号）

- 段落首行缩进：2字符（24磅）

- 报告命名规则：`【初稿/定稿】项目方名称-不良资产包收购法律尽调报告+YYYY.M.D.docx`
- 列宽参考：工商信息表3列 → 3.5cm / 7.25cm / 7.25cm

## 重要原则

- **先确认，后开工**：阶段零的四项确认全部通过前，不进入实质工作
- **表格式排版优先**：所有结构化信息（工商信息、债权明细、执行案件）均用表格呈现，不得替换为纯文本
- **保留模板结构**：原始模板的基本表述结构（六部分顺序、表格格式、用语习惯）不得改变。采用**模板克隆法**：deepcopy模板第一个债务人节的完整XML元素序列（含表格），替换文字后依次插入。不要用`add_table()`创建新表格，否则边框/底纹/列宽等格式会丢失
- **债权金额**以债权清单为准，但注明"以贷款档案资料及借款合同约定为准"
- **诉讼状态**决定时效判断："执行中""终本""待申请执行""二审待开庭" = 不存时效问题
- **破产债务人**需提示及时申报债权
- **自然人保证人**仅列姓名即可
- **重复保证人**可交叉引用，避免重复查询同一企业
- **Python技术细节**：
  - f-string中使用中文引号（""）时，f-string外层的引号必须与中文引号不同，否则会提前关闭字符串
  - 对docx表格使用 `merge()` 要在设置文本之前进行
  - 金额字段可能出现 `'-'` 等占位符，需统一处理
  - **模板克隆法**：不要用 `add_table()` 创建新表格（会丢失模板边框/底纹/列宽）。应 deepcopy 模板第一个债务人节的所有XML元素（从第二个Heading 2到第二个Heading 2），依次克隆并插入
  - **`addnext()` 顺序陷阱**：`addnext()` 在参考元素后立即插入。如果要追加多个元素，必须每次更新 `insert_after` 指针，且对每个债务人只在循环外初始化一次 `insert_after`，否则债务人顺序会反转
  - **文字跨run问题**：docx中同一段落文字可能被拆到多个 `w:r > w:t` 中（封面大字尤其常见）。`replace_para_text()` 逐run匹配，跨run文字无法替换。应使用 `set_para_text()` —— 保留首个run的 `rPr` 格式，清空其余run，一次写入全文
  - **替换顺序**：全局替换（如 `忻州农商行→中信金融资产`）会改变文字，使后续含旧文字的特定条件判断失效。**关键原则**：先用 `original_text = 全run拼接` 保存替换前原文用于条件匹配，再执行替换
  - **覆盖范围控制**：`update_non_debtor_parts()` 只能处理封面和声明段（截止到"第二部分 正 文"），绝不能遍历全文替换关键字，否则会破坏债务人模板中的文字
  - **目录处理**：TOC在docx中为 `w:sdt` 元素（`docPartGallery="Table of Contents"`），无法在Python中刷新。应清空 `sdtContent` 旧渲染条目，保留 `sdtPr` 字段结构，提示用户在Word中手动"右键→更新域"

### 数据源：企查查司法拍卖 Excel

```python
def normalize_prop(title):
    t = str(title).strip()
    t = re.sub(r'^【?(?:第一次拍卖|第二次拍卖|变卖)】?\s*', '', t)
    t = re.sub(r'^(?:一拍|二拍|变卖)\s*', '', t)
    t = re.sub(r'^(?:XXX人民法院)\s*关于\s*', '', t)
    t = re.sub(r'^位于\s*', '', t)

    t = re.sub(r'的公告$', '', t)
```

```python
def extract_room_key(prop):
    """提取房间号标识，如 '7层7086号' → ['7086']"""
    ids = re.findall(r'\d+层([\d]+[A-Za-z]*)号', prop)
    if not ids:
        ids = re.findall(r'\d+幢([\d]+[A-Za-z]*)号', prop)
    if not ids:
        ids = re.findall(r'地下\d+层([\d]+[A-Za-z]*)号', prop)
    if not ids:
        ids = re.findall(r'(\d{3,}[A-Za-z]?)号', prop)
    return sorted(set(ids))
```

### 常见陷阱

1. **"公告"行混入**：Excel 中有纯公告行（起拍价为 `-`，标题以"XXX法院关于...的公告"结尾），应用 `normalize_prop()` 统一处理，避免拆出重复条目

2. **案号变体**：同一案件可能在不同拍卖阶段使用不同案号组合（如 `(2023)晋01执恢553号` 和 `(2023)晋01执恢553号,(2022)晋01执恢553号`），匹配时需做子集判断

3. **案号缺失**：杏花岭区法院部分条目案号列为 `-`，这些按房产独立分组

4. **阶段判定遗漏**：当某条目标题无阶段关键词且日期推断规则不覆盖时，可能漏标阶段。建议在最终验证时人工复核"一拍流拍"标签数量偏少的案号组

5. **24套/21套等批量房产**：无具体房间号，只能按案号+描述文本匹配去重。注意区分不同年份的同一批量描述（如2019年和2024年分别有"24套"拍卖）

#### 第二步：拍卖阶段判定

按日期升序排列事件，分析价格走势判定阶段：

```python
def determine_stage(events):
    """判定拍卖当前阶段。
    events = [(date_str, 起拍价, 评估价), ...] 按日期升序。
    价格规律：一拍(~80%评估价) → 二拍(降价~64-72%评估价) → 变卖(同二拍价, 60天)
    """
    if not events:
        return '待查'

    # 去重（同价格+同日期的重复记录）
    unique = []
    seen = set()
    for e in events:
        key = (e[0], e[1])
        if key not in seen:
            seen.add(key)
            unique.append(e)
    events = unique

    if len(events) == 1:
        return '一拍流拍'

    prices = [e[1] for e in events]
    has_price_drop = prices[0] > prices[-1]

    # 检测60天变卖期：日期范围跨月
    has_60day = False
    for e in events:
        d = e[0]
        if '至' in d:
            parts = d.split('至')
            if len(parts) == 2:
                d1 = re.sub(r'[年月]', '-', parts[0]).replace('日', '').replace('10时', '').strip()
                d2 = re.sub(r'[年月]', '-', parts[1]).replace('日', '').replace('10时', '').strip()
                if d1[:7] != d2[:7]:  # 跨月 → 60天变卖
                    has_60day = True
                    break

    if has_price_drop and has_60day:
        return '一拍、二拍、变卖均流拍'
    elif has_price_drop and len(events) >= 2:
        return '二拍流拍'
    elif len(events) >= 2:
        return '一拍流拍'
    return '一拍流拍'
```

**判定逻辑说明**：
- 仅1条记录 → 一拍流拍
- 多条记录、价格有下降、有60天变卖期 → 一拍、二拍、变卖均流拍
- 多条记录、价格有下降、无60天变卖期 → 二拍流拍
- 多条记录、价格无下降（多次一拍） → 一拍流拍

> ⚠️ **常见错误**：不要用字符串长度判断60天变卖期（如 `len(date_str) > 25`），日期格式差异会导致误判。必须解析日期范围，判断是否跨月。

### 常见陷阱

1. **标的物归一化不完整**：公告标题可能以"XXX法院关于"开头、以"（第X次拍卖）的公告"结尾，需要用 `normalize_title()` 去除前后缀，否则同一标的物的多条记录无法合并

3. **多次一拍情形**：部分标的物可能有多条一拍记录（不同时间重新上拍），价格相同、无降价。此时应判定为"一拍流拍"而非"二拍流拍"

4. **案号相同的不同标的物**：同一案号可能涉及多个独立标的物（如机器设备和分公司机器设备），分组时必须同时使用 `(标的物, 案号)` 作为key

```python
            continue

    return header, data_rows
```

### 常见陷阱

1. **Excel 金额含千分位逗号**：`str.replace(',', '')` 后再 `float()` 转换

2. **金额为 `-` 或空**：Excel 中无效金额用 `-` 占位，需判断后转为 0

3. **模板表头不匹配**：Word 模板的表头列名和列数可能与实际需要不一致。应以 Excel 数据为准，用 `set_cell_font()` 修正表头文字

4. **终本案件需合计行**：表格末尾追加一行"合计"，合并前几列为空，仅在执行标的和未履行金额列显示汇总数

5. **失信表缺少列**：企查查失信 Excel 没有"执行依据文号"和"履行情况"列，这两列分别填写"待查"和"全部未履行"

6. **段落中的汇总数字**：引导段落中可能包含汇总数字（被执行总金额 XX 万元），需要在重建表格后同步更新

7. **模板残留文字**：模板可能从其他项目复制而来，残留其他公司的名称等信息，需要全文检查并替换

8. **读取 Excel 时的表头检测**：不同 Excel 的表头行位置可能不同，应使用 `vals[0] == '序号'` 动态检测表头行，不要硬编码行号

9. **docx 保存后表格样式**：`table.add_row()` 添加的行会继承表格默认样式，但字体等格式需要单独设置。确保每个单元格都通过 `set_cell_font()` 设置仿宋字体

10. **限高 Excel 与 Word 列顺序不同**：企查查限高 Excel 列顺序为 `序号|案号|限消令对象|限制法定代表人|申请人|涉案金额|执行法院|立案日期|发布日期`，而 Word 表格列顺序为 `序号|案号|执行法院|限制法定代表人|申请人|涉案金额|立案日期|发布日期`。注意 `执行法院` 在 Excel 第7列(索引6)、在 Word 第3列(索引2)，需要重新映射

11. **限消令对象恒为同一公司**：限高 Excel 中"限消令对象"列恒为目标公司名称，在 Word 表格中可省略此列

12. **表头底纹 tint 值**：用 `doc.add_table()` 新建表格时，设置 `fill=ADB9CA, themeFill=text2` 还不够，必须同时设置 `themeFillTint='66'`，否则底纹颜色深浅与模板不一致。模板克隆法则无此问题

13. **模板克隆优于从零创建**：不要用 `doc.add_table()` 从零建表，否则边框（外双线内单线）、底纹、列宽、单元格边距（tblCellMar）、表格居中（jc）等全部丢失。始终复制模板文件再修改内容

14. **企查查 Word 报告表索引定位**：企查查 Word 报告包含 60+ 个表格，不能靠猜测索引定位。必须通过表头内容匹配找到正确的表。遍历所有表格的 header row，根据关键词（被执行人、失信被执行人、终本案件、限制高消费）匹配。数据分"当前"和"历史"两部分，分布在不同的表索引位置，需要分别提取后合并

15. **企查查 Word 报告限高表无执行法院列**：与 Excel 导出不同，企查查 Word 报告的限制高消费表没有"执行法院"列。不要直接填"待查"——应从 Table 15（司法案件）中按案号交叉匹配获取法院。需注意案号格式差异（旧式`2015年小执字第00545号` vs 新式`（2015）小执字第545号`），匹配前先规范化案号

16. **仅使用当前数据，不合并历史数据**：企查查 Word 报告的执行案件信息分"当前"（4.2-4.5节）和"历史"（11.7-11.9节）两部分。**报告只应使用当前数据**，不应合并历史数据。历史数据是企查查归档的已失效记录（如已履行完毕的执行案件、已解除的限高等）。

19. **主报告内执行案件表与独立文档格式不同**：尽调主报告中的被执行案件表（在(三)主体资格审查的第(6)小节内）使用网格型6样式、ADB9CA+tint=66表头、thinThickSmallGap边框(sz=12)，含合计行。这与独立生成的"被执行案件信息.docx"格式一致，但嵌入位置在主体资格审查节内，每个债务人+每个担保人各自独立。

20. **对外投资仅读取企业当前投资，勿混入法定代表人个人信息**：企查查报告中"对外投资"表和"法定代表人对外投资"表是两个不同的表，**表索引因报告而异，不可硬编码**。区分方法不看索引，看列名：

    - 同理，「担任法定代表人起止时间」→ 法定代表人对外投资持股；「任职起止时间」→ 法定代表人任职信息，均非企业对外投资

21. **股东和董监高信息嵌入在工商登记表内部**：模板的工商登记表（3列Table Grid）不是简单的字段-值对，末尾嵌入了"董事、监事、高级管理人员信息"和"股东信息"两个子节（各有子表头行+数据行）。从企查查 Table 3（股东信息）和 Table 4（主要人员）提取数据后，按此子表格式填入。

22. **司法拍卖表底纹颜色不同于执行案件表**：司法拍卖表表头使用 **D9E2F3**（浅蓝色）填充，而执行案件表使用 **ADB9CA+tint=66**（蓝灰色）。两者不可混用。模板克隆法自动保留正确颜色，无需手动设置。

23. **对外投资/动产抵押/股权质押 Qichacha列映射差异**：企查查Word报告与模板的列结构不完全对应：
    - 对外投资：Qichacha（8列）→ 模板（7列），**缺少"法定代表人"和"注册时间"**，填"待查"；"投资数额"用"注册资本"代理
    - 动产抵押：Qichacha（8列）→ 模板（5列），跳过"状态""抵押人""债务履行期限"
    - 股权质押：Qichacha（8列）→ 模板（6列），**跳过"出质股权标的企业"**（Qichacha col 3），质权人在 col 4 而非 col 3

24. **Qichacha金额字段可能带"万元"后缀**：动产抵押的"被担保主债权数额"和股权质押的"出质股权数额"在企查查报告中可能以"万元""万人民币元""元"结尾。填入模板前必须用 `clean_amount()` 清洗，因为模板列名已含"（万元）"。清洗逻辑：`str(val).replace(',', '').replace('万元', '').replace('万人民币元', '').replace('元', '').strip()`。

```python
from docx import Document

report = Document('企查查报告.docx')

def extract_table(table):
    """提取表格所有行"""
    rows = []
    for row in table.rows:
        cells = [c.text.strip().replace('\n', ' ') for c in row.cells]
        rows.append(cells)
    return rows

def find_tables_by_header(doc, keyword):
    """根据表头关键词匹配表索引"""
    matches = []
    for i, t in enumerate(doc.tables):
        hdr = [c.text.strip()[:20] for c in t.rows[0].cells]
        if any(keyword in c for c in hdr):
            matches.append(i)
    return matches
```

#### 限高法院字段交叉匹配

企查查 Word 报告的限高表中没有"执行法院"列，需要从 Table 15（司法案件）中交叉匹配获取：

```python
# 1. 从 Table 15（司法案件）构建 案号→法院 映射表
sf_data = extract_table(report.tables[15])
court_map = {}
for row in sf_data[1:]:
    case_no = row[5]   # 案号在 Table 15 的索引5
    court = row[6]     # 法院在 Table 15 的索引6
    if case_no and case_no != '-':
        court_map[case_no] = court

# 2. 案号规范化：旧式案号格式不同，需统一
#    限高表: "2015年小执字第00545号" → 司法案件表: "（2015）小执字第545号"
def normalize_case_no(case_no):
    cn = re.sub(r'字第0+', '字第', case_no)      # 去前导零
    cn = re.sub(r'^(\d{4})年', r'（\1）', cn)     # 年→括号格式
    return cn

# 3. 匹配（先直接匹配，再规范化后匹配）
def get_xg_court(case_no):
    if case_no in court_map:
        return court_map[case_no]
    norm = normalize_case_no(case_no)
    if norm in court_map:
        return court_map[norm]
    return '待查'
```

### 司法案件（合并数据源）场景

#### 适用情况

#### 司法案件 Excel 结构

文件名：`【企查查】司法案件-XX公司(XXX).xlsx`

Sheet 名：`司法案件`

#### 新增表格时的格式修复

如果用 `doc.add_table()` 新增了表格（而非从模板克隆），需手动修复格式以匹配模板：

```python
from docx.oxml import OxmlElement
from docx.oxml.ns import qn
from docx.enum.text import WD_ALIGN_PARAGRAPH
from copy import deepcopy

def fix_table_format(new_table, ref_table):
    """将 new_table 的格式修复为与 ref_table 一致"""
    # 1. 克隆表格样式和边框
    new_table.style = ref_table.style
    tblPr_ref = ref_table._tbl.find(qn('w:tblPr'))
    tblPr_new = new_table._tbl.find(qn('w:tblPr'))
    if tblPr_new is None:
        tblPr_new = OxmlElement('w:tblPr')
        new_table._tbl.insert(0, tblPr_new)
    
    # 替换边框
    ref_borders = tblPr_ref.find(qn('w:tblBorders'))
    if ref_borders is not None:
        existing = tblPr_new.find(qn('w:tblBorders'))
        if existing is not None:
            tblPr_new.remove(existing)
        tblPr_new.append(deepcopy(ref_borders))
    
    # 克隆 tblCellMar, tblLayout, jc（单元格边距、布局、居中）
    for tag in ('w:tblCellMar', 'w:tblLayout', 'w:jc', 'w:tblLook'):
        ref_elem = tblPr_ref.find(qn(tag))
        if ref_elem is not None:
            existing = tblPr_new.find(qn(tag))
            if existing is not None:
                tblPr_new.remove(existing)
            tblPr_new.append(deepcopy(ref_elem))

    # 2. 表头行：底纹 + 加粗 + 居中 + 垂直居中
    for ci, cell in enumerate(new_table.rows[0].cells):
        tcPr = cell._tc.get_or_add_tcPr()
        # 替换（非追加）底纹元素
        old_shd = tcPr.find(qn('w:shd'))
        if old_shd is not None:
            tcPr.remove(old_shd)
        shading = OxmlElement('w:shd')
        shading.set(qn('w:val'), 'clear')
        shading.set(qn('w:color'), 'auto')
        shading.set(qn('w:fill'), 'ADB9CA')
        shading.set(qn('w:themeFill'), 'text2')
        shading.set(qn('w:themeFillTint'), '66')  # 必须，否则颜色深浅不一致
        tcPr.insert(0, shading)
        
        # 垂直居中
        old_va = tcPr.find(qn('w:vAlign'))
        if old_va is not None:
            tcPr.remove(old_va)
        vAlign = OxmlElement('w:vAlign')
        vAlign.set(qn('w:val'), 'center')
        tcPr.append(vAlign)
        
        # 水平居中 + 加粗
        cell.paragraphs[0].alignment = WD_ALIGN_PARAGRAPH.CENTER
        for run in cell.paragraphs[0].runs:
            run.font.bold = True

    # 3. 数据行：垂直居中
    for ri in range(1, len(new_table.rows)):
        for ci, cell in enumerate(new_table.rows[ri].cells):
            tcPr = cell._tc.get_or_add_tcPr()
            if tcPr.find(qn('w:vAlign')) is None:
                vAlign = OxmlElement('w:vAlign')
                vAlign.set(qn('w:val'), 'center')
                tcPr.append(vAlign)
```

- P0: 标题（`XX公司的被执行案件信息`）
- P1: `（6）被执行案件情况：`
- P2: 被执行人引导段落（含金额汇总）
- Table 0: 被执行人表（5列：序号|案号|执行标的（元）|执行法院|立案日期）
- P3: 终本案件引导段落（如有）
- Table 1: 终本案件表（7列，末尾含合计行）
- P4: 失信被执行人引导段落（如有）
- Table 2: 失信被执行人表（6列：序号|案号|执行法院|执行依据文号|履行情况|立案日期）

### 陷阱25：基础债权表格格式转换 — 扁平标签格式 vs 黄金标准

部分旧模板的基础债权表使用**扁平标签格式**（10行），需转换为**黄金标准格式**（8行）：

**扁平格式（10行，错误）：**
```
Row 0: 借款合同编号|借款人|借款金额|本金余额|借款起止日期|合同日期
Row 1: loan data
Row 2: 保证人|<value>|""|""|""|""  ← 单行扁平标签
Row 3: 抵质押物|<value>|""|""|""|""  ← 单行扁平标签
Rows 4-9: "待查" 填充行 (带vMerge/gridSpan)
```

**黄金标准（8行，正确）：**
```
Row 0: 借款合同编号|借款人|借款金额|本金余额|借款起止日期|合同日期
Row 1: loan data
Row 2: 保证合同编号|保证方式|保证人|担保金额（元）|保证期间|合同日期  ← 子表头
Row 3: 待查|待查|<保证人>|待查|待查|待查  ← 数据行
Row 4: 抵押合同编号|抵押类型|抵押人|抵押物|担保金额（元）|合同日期  ← 子表头
Row 5: 待查|<类型>|<抵押人>|<抵押物>|待查|待查  ← 数据行
Row 6: 待查|抵押物权证号|他项权证号|地理位置|登记日期|抵押物权证号  ← 子表头
Row 7: 待查|待查|待查|<地理位置>|待查|待查  ← 数据行
```

**转换要点：**
- 保证人/抵质押物数据从扁平格式Row 2-3提取，从Excel债权清单补充
- 无保证人时第3行写"无"而非"待查"
- 抵押类型从抵质押物描述自动检测：`土地/房产/房屋/商用楼/工业用地` → 不动产，`设备/机器/镍铁` → 动产
- 地理位置从抵质押物描述中提取（正则`位于(.+?)的`）
- 抵押人名称从"XX公司名下"或"XX公司以其持有的"模式提取

### 陷阱30：IC表（工商登记信息表）重建时tcBorders全部丢失 【严重 — 2026.5自查发现】

`rebuild_ic_table()` 使用 `table.add_row()` 创建所有新行，但 `add_row()` 创建的 `<w:tr>` 不含 `<w:tcBorders>` 子元素。原始IC表的每一行都有完整的单元格边框定义（thinThickSmallGap/sz=12 outer + single/sz=4 inner），重建后 Row 1 ~ N 全部没有 tcBorders。

**错误写法：**
```python
def rebuild_ic_table(table, ...):
    # Row 0 留在原位 → tcBorders 保留
    while len(table.rows) > 1:
        table._tbl.remove(table.rows[1]._tr)
    # 用 add_row() 添加新行 → 无 tcBorders！
    r = table.add_row()
```

**正确做法：** 不要用 `add_row()` 重建。保留原始行元素，只替换文本内容。或者使用 `deepcopy(original_row._tr)` 克隆行元素，保留其所有 `tcPr` 子元素（tcBorders, tcW, shd, vAlign 等）。

**自查方法：** 遍历原始和更新文档的对应IC表，逐行检查 `cell._tc.find(qn('w:tcPr')).find(qn('w:tcBorders'))` 是否为 None。

### 陷阱31：股东"认缴出资"列映射误匹配日期列 【严重 — 2026.5自查发现】

企查查股东信息表表头典型顺序：
```
序号 | 股东名称 | 持股比例 | 认缴出资额(万元) | 认缴出资日期 | 首次持股日期
```

列匹配代码 `'认缴出资' in c` 会同时匹配 **"认缴出资额(万元)"** 和 **"认缴出资日期"**。由于 `in` 是子串匹配，两个列名都包含"认缴出资"，循环中后匹配到的列会覆盖前面的，导致 `col_cap` 最终指向日期列而非金额列。

**症状：** 股东认缴出资显示日期（如"2015-04-28"）而非金额（如"1875"）。

**正确写法：**
```python
if c and '认缴出资额' in c and '日期' not in c:
    col_cap = ci
```
或者用精确匹配：
```python
if c and c.startswith('认缴出资额'):
    col_cap = ci
```

**教训：** 企查查所有列名匹配必须精确，优先用 `==` 或 `startswith`，禁止用模糊的 `in` 子串匹配。

### 陷阱36：table.add_row() 的通用陷阱 —— 所有XML属性都会丢失 【2026.5自查总结】

`python-docx` 的 `table.add_row()` 是**最大破坏性操作**。它创建一个全新的空 `<w:tr>` 元素，不带任何以下属性：

| 丢失的属性 | 影响 |
|-----------|------|
| `w:tcBorders` | 单元格边框消失，表格看起来"透光" |
| `w:tcW` (特殊宽度) | 合并单元格效果丢失，子表头行变窄 |
| `w:shd` | 单元格底纹/背景色消失 |
| `w:vAlign` | 垂直居中丢失 |
| `w:gridSpan` | 跨列合并丢失 |

**铁律：凡是需要保留表格外观的重建操作，必须使用 deepcopy 原始行元素，只替换文本内容（`w:t` 元素），绝不用 `add_row()`。**

```python
# 推荐模式：克隆模板行，只替换文本
from copy import deepcopy
template_row_tr = table.rows[1]._tr  # 保留一行作为模板
for data in data_rows:
    new_tr = deepcopy(template_row_tr)  # 克隆全部XML属性
    t_elems = new_tr.findall('.//' + qn('w:t'))
    for ti, t_elem in enumerate(t_elems):
        t_elem.text = str(data[ti]) if ti < len(data) else ''
    table._tbl.append(new_tr)
```

此陷阱是陷阱30、31、32、34的共同根因。

### 陷阱40：企查查表匹配必须 first-match-wins + 精确条件防误覆盖 【严重 — 2026.5发现】

```python

    # ✅ 正确：加 guard + 精确条件
```

### 陷阱43：独立文档所有表格外框必须为双线(thinThickSmallGap)，内框为单线(single) 【严重 — 2026.5发现】

参考模板（{{公司}}的被执行案件信息.docx）所有表格的边框规格：

| 边框位置 | val | sz | color |
|---------|-----|-----|-------|
| 外框(top/left/bottom/right) | `thinThickSmallGap` | 12 | auto |
| 内框(insideH/insideV) | `single` | 4 | auto |

**错误做法**：使用 `doc.add_table()` 直接创建表格，默认边框全部为 `single, sz=4`（内外同粗细单线），与模板的"外双线内单线"不一致。

**正确做法**：所有用 `add_table()` 创建的表格，必须在创建后手动设置 `tblBorders` 元素：

```python
from docx.oxml import OxmlElement
from docx.oxml.ns import qn

def set_table_borders(table):
    """设置表格边框：外框双线(thinThickSmallGap, sz=12)，内框单线(single, sz=4)"""
    tbl = table._tbl
    tblPr = tbl.find(qn('w:tblPr'))
    if tblPr is None:
        tblPr = OxmlElement('w:tblPr')
        tbl.insert(0, tblPr)
    
    # 移除旧边框
    old_borders = tblPr.find(qn('w:tblBorders'))
    if old_borders is not None:
        tblPr.remove(old_borders)
    
    borders = OxmlElement('w:tblBorders')
    
    # 外框：双线 thinThickSmallGap, sz=12
    for border_name in ('top', 'left', 'bottom', 'right'):
        border = OxmlElement(f'w:{border_name}')
        border.set(qn('w:val'), 'thinThickSmallGap')
        border.set(qn('w:sz'), '12')
        border.set(qn('w:space'), '0')
        border.set(qn('w:color'), 'auto')
        borders.append(border)
    
    # 内框：单线 single, sz=4
    for border_name in ('insideH', 'insideV'):
        border = OxmlElement(f'w:{border_name}')
        border.set(qn('w:val'), 'single')
        border.set(qn('w:sz'), '4')
        border.set(qn('w:space'), '0')
        border.set(qn('w:color'), 'auto')
        borders.append(border)
    
    tblPr.append(borders)

def set_table_jc_center(table):
    """设置表格居中"""
    tblPr = table._tbl.find(qn('w:tblPr'))
    if tblPr is None:
        tblPr = OxmlElement('w:tblPr')
        table._tbl.insert(0, tblPr)
    old_jc = tblPr.find(qn('w:jc'))
    if old_jc is not None:
        tblPr.remove(old_jc)
    jc = OxmlElement('w:jc')
    jc.set(qn('w:val'), 'center')
    tblPr.append(jc)
```

**自查方法**：打开生成的文档，检查任意表格的外边框是否为双线（比内线更粗），内边框是否为单线。
