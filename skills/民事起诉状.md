## 功能说明

本 Skill 用于处理**民事起诉状起草**事务。以原告代理律师的工作标准，基于案件目录内的真实材料起草民事起诉状，输出符合法院立案要求的 .docx 文书。

**上游输入**：优先读取案件目录「文书/」下的《卷宗梳理工作底稿》（由「卷宗梳理」Skill 产出）；无底稿的，先按卷宗梳理的纪律快速通读材料再起草。

**定位**：起诉状是请求权的正式表达，每一个字都可能被对方在答辩和质证中利用。本 Skill 的核心价值不是"填模板"，而是确保**每个主张有证据、每个金额有算式、每个管辖有依据**。

---

## 一、起草纪律（最高优先级）

1. **事实主张必须对应证据**：起诉状"事实与理由"部分的每一个事实主张，必须能对应案件材料中的具体证据；起草时在工作稿中为每个事实主张标注来源文件名，正式稿中通过"证据和证据来源"部分体现对应关系。材料中没有证据支持的事实，要么删除，要么列入待补信息清单。
2. **请求金额必须给计算明细**：本金的构成、利息的起算日与利率依据、违约金的条款出处，逐项写明计算式。"暂计至起诉之日"的，写明暂计截止日和后续主张方式。
3. **管辖法院必须给依据**：有约定管辖条款的，引用条款原文及其在合同中的位置；无约定的，做法定管辖分析并写明结论依据。
4. **关键信息缺失时输出待补信息清单，禁止编造**：被告身份证号不明、利率约定不清、住所地待查等情形，一律在文书末尾附「待补信息清单」，对应位置用【待补：××】占位，绝不虚构。
5. **草稿标注**：文书首页页眉或文末必须标注"**草稿·须律师审核后使用**"；文件名含"草稿"字样。
6. **输出落盘**：写入案件目录「文书/」子目录（不存在则创建）。
7. **语体**：正式法律文书语体，禁用 emoji，禁用网络用语和情绪化表述。

## 二、核心工作流

### Step 0：读取上游材料

- 读取《卷宗梳理工作底稿》；核对其中「待核实」事项是否已清零，未清零的评估是否阻断起草（阻断的先输出待补信息清单并停止）
- 无底稿的，按卷宗梳理 Skill 的通读顺序快速阅卷，自建简版时间轴和证据对应表

### Step 1：确定案由与请求权基础

- 从法律关系定性出发选定案由（参照最高人民法院《民事案件案由规定》，以现行有效版本为准，引用前核对）
- 明确每项请求的请求权基础（合同条款/法律规定），写入工作稿备查——这决定"事实与理由"要证明哪些要件事实

### Step 2：设计诉讼请求

- 逐项列明，每项独立可执行（金钱给付/行为/确认/解除各自单列）
- 金钱请求按 §四 的计算规范出明细
- 末项固定加列诉讼费用负担请求

### Step 3：管辖分析

- 按 §五 的方法确定受诉法院，形成管辖依据说明（写入工作稿，正式稿中体现为"此致"法院的选择）

### Step 4：撰写事实与理由

- 按时间顺序叙述要件事实：关系成立 → 己方履行 → 对方违约 → 损失/欠付现状 → 催告与协商经过
- 每个事实主张在工作稿中标注证据来源；叙事只写能证明的事实，与请求无关的背景一笔带过或不写

### Step 5：编制证据和证据来源部分

- 与「证据清单」Skill 衔接：已有规范证据清单的直接概括引用；没有的，在起诉状中按"证据名称+证据来源+证明内容"简列（详细清单另行制作）

### Step 6：组装与格式化输出

- 按 §三 结构组装全文，按 §六 格式规范生成 .docx
- 末尾附「待补信息清单」（如有）
- 文件命名：`民事起诉状（草稿）_【原告简称】诉【被告简称】_YYYYMMDD.docx`

## 三、起诉状结构与逐部分起草规范

> 结构依据《民事诉讼法》第124条关于起诉状记明事项的要求（以现行有效版本为准，引用前核对）；可参考最高人民法院发布的民事起诉状示范文本样式。

### 3.1 标题

「民事起诉状」，居中。

### 3.2 当事人信息块

**自然人**（原告、被告均按此）：

```
原告：姓名，性别，XXXX年XX月XX日出生，X族，住址：XXX，联系方式：XXX，
公民身份号码：XXX。
```

**法人或非法人组织**：

```
被告：XXX有限公司，住所地：XXX，统一社会信用代码：XXX。
法定代表人：XXX，职务：XXX。联系方式：XXX。
```

信息块纪律：
- 名称必须与营业执照/身份证一字不差，从材料中的登记信息或合同签署页核对
- 任一字段材料中查不到的，用【待补：××】占位并记入待补信息清单
- 多名当事人按"原告一、原告二/被告一、被告二"编号；第三人单独列明

### 3.3 诉讼请求

逐项编号，每项一个独立请求。金钱请求示例写法：

```
一、判令被告向原告支付货款人民币486,000元；
二、判令被告向原告支付逾期付款违约金（以486,000元为基数，按《工业品买卖合同》
第9条约定的日万分之五标准，自2024年10月1日起暂计至2026年6月10日为149,445元，
此后计至实际清偿之日止）；
三、判令被告承担本案诉讼费用。
```

每项金钱请求在工作稿中必须附 §四 格式的计算明细。

### 3.4 事实与理由

推荐五段式（按案情裁剪）：

1. **法律关系成立**：何时、何方式订立合同/形成关系，核心约定（标的、金额、期限、违约条款——引用条款号）
2. **原告履行情况**：已履行什么，有何凭据
3. **被告违约事实**：未付款/未履行的具体情况，违约起算点
4. **欠付/损失现状**：截至起诉时的金额构成
5. **催告与协商经过**：催告事实（兼具时效中断说明的作用），协商未果，故诉至法院

理由部分点明请求权基础：引用法律名称+条款号（如《民法典》第577条、第579条），并遵循"以现行有效版本为准，引用前核对"的纪律，不展开学理论述。

### 3.5 证据和证据来源

```
证据一：2024年3月12日《工业品买卖合同》，来源：原告留存原件，证明双方买卖
合同关系成立及货款金额、付款期限、违约金条款的约定；
证据二：银行转账回单3张，来源：原告开户行打印，证明原告已付款项及被告欠付
金额的计算基础；
……
```

### 3.6 此致法院

```
此致
XX市XX区人民法院
```

受诉法院按 §五 的管辖分析结论填写；工作稿中保留管辖依据说明备律师审核。

### 3.7 落款

```
                                            具状人：（签名/盖章处留空）
                                            日期：    年    月    日（留空）
```

落款日期一律留空，由律师确认提交日时填写。

## 四、诉讼请求金额计算规范

每项金钱请求附计算明细块（工作稿必备，正式稿视篇幅放入诉讼请求括注或事实与理由部分）：

```
【本金】
应付总额 1,200,000 元（《工业品买卖合同》第3条）
已付 714,000 元（银行回单：2024.6.20付500,000元；2024.8.15付214,000元）
欠付本金 = 1,200,000 - 714,000 = 486,000 元

【违约金】
条款依据：《工业品买卖合同》第9条"每逾期一日按未付款项的万分之五支付违约金"
起算日：2024年10月1日（第9条约定付款期限届满次日）
暂计止：2026年6月10日（起诉前暂计日），共618日
违约金 = 486,000 × 0.05% × 618 = 150,174 元
（注：日万分之五年化约18.25%，存在被法院调减的可能，已向委托人提示）

【利息类请求的利率依据写法】
有约定 → 引用条款号及约定利率；
无约定 → 主张按全国银行间同业拆借中心公布的贷款市场报价利率（LPR）计算逾期
利息（依据《民法典》第511条、第676条相关规则，以现行有效版本为准，引用前核对）。
```

计算纪律：
- 每个数字可溯源：本金对应合同与付款凭证、起算日对应合同条款或催告文件、利率对应条款或法定标准
- 大小写金额并用时核对一致；计算结果人工复核一遍加减乘除
- 材料中金额相互矛盾（如对账单与合同不一致）且未经委托人确认的，**禁止择一径直写入**，列入待补信息清单
- 违约金明显偏高（参考超过年化 LPR 四倍或合同编通则司法解释关于损失30%的调减规则，以现行有效版本为准）的，按约定主张但在工作稿中提示调减风险

## 五、管辖法院分析方法

### 5.1 分析顺序（先约定后法定）

1. **检查专属管辖**：不动产、港口作业、继承等纠纷适用专属管辖（《民事诉讼法》第34条，以现行有效版本为准，引用前核对），有专属管辖的，约定无效
2. **检查仲裁条款**：合同含有效仲裁条款的，法院无管辖权——发现仲裁条款立即停止起诉状起草，向用户提示应走仲裁程序
3. **检查约定管辖**：有书面管辖协议的，核查其效力（与争议有实际联系的地点、不违反级别管辖和专属管辖——《民事诉讼法》第35条），引用条款原文及位置
4. **法定管辖**：无有效约定的，合同纠纷由被告住所地或合同履行地法院管辖（《民事诉讼法》第24条）；合同履行地的确定参照《民诉法解释》关于争议标的为给付货币时接收货币一方所在地等规则（以现行有效版本为准，引用前核对）
5. **级别管辖**：按标的额和当地标准确定基层/中级法院

### 5.2 管辖依据说明模板（写入工作稿）

```
管辖结论：XX市XX区人民法院
依据：《工业品买卖合同》第12条约定"因本合同发生的争议，由原告所在地人民法
院管辖"。原告住所地位于XX市XX区，该约定不违反级别管辖和专属管辖规定，应属
有效（《民事诉讼法》第35条，以现行有效版本为准，引用前核对）。
风险提示：（如有，例如约定地点与争议实际联系存疑）
```

使用 python-docx 生成，遵循以下规范：

| 要素 | 规范 |
|------|------|
| 纸张与页边距 | A4；上下 2.54cm、左右 3.18cm |

## 七、待补信息清单机制

正文中以【待补：××】占位的事项，统一汇总到文书末尾（独立分页或独立区块）：

```
——以下为内部工作内容，提交法院前删除——

待补信息清单
1.【待补：被告住所地】材料中仅有合同载明的地址（2024年），需以工商登记现状
  为准，建议打印国家企业信用信息公示系统报告。
2.【待补：第二批货物签收日期】《送货单（第二批）》签收日期模糊，影响违约金
  起算的事实陈述，建议调取原件或以催告函送达日起算（需委托人确认）。
3.【待补：原告开户行付款回单（2024.8.15）】卷内仅有截图，立案需打印盖章版。
```

清单纪律：每项写明"缺什么、影响起诉状哪个部分、建议怎么补"；存在阻断性缺失（如被告名称无法确认）时，停止输出"完整"起诉状，改为输出结构框架+待补清单，并明确告知用户。

## 八、输入→输出示例片段

**输入**（卷宗梳理底稿时间轴节选）：

```
2024-03-12 | 甲乙签订《工业品买卖合同》，总价120万元 | 依据：合同第3条
2024-06-20 | 甲方付款50万元 | 依据：银行回单
2024-09-30 | 付款期限届满（合同约定货到验收后30日付清） | 依据：合同第5条
2025-01-10 | 乙方微信催款，甲方回复"年后处理" | 依据：微信记录第18页
```

**输出**（事实与理由对应段落）：

```
二、原告已依约履行供货义务，被告未按期支付货款
2024年3月12日，原告与被告签订《工业品买卖合同》，约定货款总额1,200,000元，
被告应于货到验收后30日内付清（合同第3条、第5条）。原告交付货物并经被告验收
后，被告仅于2024年6月20日支付500,000元，余款486,000元至今未付。2025年1月10
日原告通过微信催告，被告答复"年后处理"，此后仍未付款。
```

每句事实均可回溯到时间轴中带出处的节点——这是合格输出的判定标准。

## 十一、注意事项

1. **起诉状是给法官看的，不是给委托人出气的**：删除一切情绪化、评价性表述（"恶意拖欠""毫无诚信"），用事实和数字说话。
2. **请求宁可精确勿求其大**：明显无依据的高额请求会推高诉讼费并损害整体可信度；拿不准的损失项，提示委托人补证后另行主张或在举证期限内变更诉请。
3. **时效自查**：起草时核对最后一次催告/还款时间，距今接近三年的在工作稿中显著提示（《民法典》第188条，以现行有效版本为准）。
4. **多被告的责任形态写明**：连带/按份/补充责任在诉讼请求中明确表述，并有请求权基础支撑。
5. **与证据清单联动**：起诉状定稿前应同步生成规范证据清单（用「证据清单」Skill），两者的证据名称、编号、证明目的必须一致。
6. **不代替律师签发**：本 Skill 产出永远是草稿；提交法院前的最终审核、签字、日期填写由执业律师完成。
