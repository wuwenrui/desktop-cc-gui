真实触发点 输入区发现未完成请求后显示“去回答”。点击后走 onJumpToUserInputRequest， 再调用 focusUserInputRequestCard 定位请求卡片。
旧问题 scrollIntoView 是浏览器原生定位。它不只处理纵向滚动，也可能为了让元素完整可见而改变横向滚动。
新方案 手动计算请求卡片的纵向居中位置，调用消息容器 scrollTo({ left: 0, top })， 然后再次写入 scrollLeft = 0 防止横向残留。
