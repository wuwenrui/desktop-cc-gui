## 5. Daemon vs Tauri `skills_list` call relationship

Both call the **same** core:
- Tauri path: `src-tauri/src/codex/mod.rs:1543-1596` -> `crate::skills::skills_list_local_for_workspace` -> `skills::skills_list_local_core` (`src-tauri/src/skills.rs:447`).
- Daemon path: `src-tauri/src/bin/cc_gui_daemon/daemon_state.rs:2618-2660` -> `skills::skills_list_local_core` directly.
- Fallback path: `codex_core::skills_list_core` (`src-tauri/src/shared/codex_core.rs:1433-1446`).

Both build the same `Vec<Value>` JSON shape:
- Daemon (`daemon_state.rs:2631-2640`): `json!({ name, path, source, description, enabled: true })`.
- Tauri (`codex/mod.rs:1565-1574`): identical shape.

**Curated change**: extend `skills_list_local_core` signature to take `&AppSettings`; pass from
both Tauri and daemon callers; merge curated entries; flip `enabled` based on
`enabled_curated_skill_ids`. Fallback `codex_core::skills_list_core` keeps `enabled: true`
default (placeholders for next change).
