## 验证目标

按 design §1 / §3 定义的 UX 体验预算, 在一次真实工具调用 turn 上验证:

1. **文件树点击响应 < 200ms** — Codex 在 `ccgui` 仓库上的 `codex "先用这个当前项目最新代码分析"` turn 期间, 手动点击侧栏文件树节点的首帧渲染延迟.
2. **折叠栏 (Sidebar/Inspector) 点击响应 < 200ms** — 折叠/展开动画首帧延迟.
3. **切会话响应 < 200ms** — 14 个并发会话中, 切换活跃会话时新消息列表首帧延迟.

## 复现步骤

1. 启动 `feature/v0.5.14` Tauri release build.
2. 打开 14 个并发 Codex 会话 (任意 workspace).
3. 任一会话跑 `codex "先用这个当前项目最新代码分析"` 一次工具调用 turn.
4. 在 turn 持续期间, 用 DevTools Performance panel 录 30s 时间窗:
   - 手动点击侧栏文件树节点 × 3 次.
   - 手动折叠/展开侧栏 × 3 次.
   - 手动切换活跃会话 × 3 次.
5. 量测各交互的 "Input event → next paint" 时间, 记录到本文件.
